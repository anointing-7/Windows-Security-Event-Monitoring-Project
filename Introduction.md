# Introduction:
In a Security Operations Center (SOC), security events are monitored and investigated in order to identify malicious behavior and contain breaches before attackers can cause damage. This SOC Investigation Project was carried out to simulate the responsibilities of a SOC analyst in detecting and investigating security incidents. It involved the generation of Windows Event logs, which were later used to create and investigate alerts.

## Objectives:
The objectives were:
1. To configure a Windows server and an endpoint.
2. To convert the Windows server into a Domain Controller.
3. To create admin and normal user accounts using the DC.
4. To enable Windows logging and auditing.
5. To monitor Windows Event logs using Splunk.
6. To generate and investigate security alerts.
   
   ## Architecture:
A single server (Domain Controller) with a single endpoint connected to it. 

## Tools used:
-Server Manager.
-Local Group Policy.
-PowerShell.
-Splunk Enterprise.
-Splunk Universal Forwarder.
-Sysmon.
